<task type="auto">
  <name>Task 2: Generate app icons from logo</name>
  <files>packages/mobile/assets/icon.png, packages/mobile/assets/adaptive-icon.png, packages/mobile/assets/favicon.png</files>
  <action>
The draugar-logo.png is already suitable as an app icon (square with rounded corners, clear iconography).

1. Copy logo as main icon (1024x1024 for iOS):
   - Check logo dimensions first
   - If not 1024x1024, resize using ImageMagick or similar

2. Create adaptive-icon.png for Android:
   - Android adaptive icons need the foreground on transparent background
   - Since our logo has a black background that matches our brand, keep it as-is
   - Copy same as icon.png

3. Create favicon.png (48x48 for web):
   - Resize from logo

Use convert (ImageMagick) if installed, otherwise use sharp via Node.js script.

Check if ImageMagick is available:
```bash
which convert || echo "Need to use Node.js sharp"
```

If ImageMagick available:
```bash
# Get logo dimensions
identify draugar-logo.png

# Create icon.png (1024x1024)
convert draugar-logo.png -resize 1024x1024 packages/mobile/assets/icon.png

# Create adaptive-icon.png (same)
cp packages/mobile/assets/icon.png packages/mobile/assets/adaptive-icon.png

# Create favicon (48x48)
convert draugar-logo.png -resize 48x48 packages/mobile/assets/favicon.png
```

If ImageMagick not available, install sharp and resize:
```bash
cd packages/mobile
pnpm add -D sharp
```

Then create a quick script to resize.
  </action>
  <verify>
ls -la packages/mobile/assets/*.png shows updated timestamps
identify packages/mobile/assets/icon.png shows 1024x1024 (if ImageMagick)
  </verify>
  <done>App icons generated: icon.png (1024x1024), adaptive-icon.png, favicon.png (48x48)</done>
</task>
